Why another OCR option? Why like this?

This interface provides access to cloud OCR services without requiring coding or cloud-storage setup. Both supported providers accept large multi-page PDFs and deliver high accuracy on Indic scripts — unlike Google Drive OCR, which can be confused by a low-quality text layer already embedded in the file. More background here{{ ext_link_icon() }}.

Two providers are currently supported:

  • Google Cloud Vision — mature, widely used, strong general OCR. Requires a Google Cloud account with billing enabled. Setup is involved but only needs to be done once.
  • Sarvam Vision — Indian AI focused on Indic languages, including Sanskrit. Simpler to set up: just create an account at sarvam.ai{{ ext_link_icon() }} and generate a key.

Why do I need an API key? How do I get one?

Both providers are paid services that bill based on usage. Each user supplies their own API key — a password-like string linked to your own account — so you pay directly and no one else's usage is charged to you. You can create or delete a key at any time.

1b. Google Cloud Vision — written walkthrough
  1. Go to the Google Cloud Console:
    While logged into your Google account, visit the Google Cloud Console{{ ext_link_icon() }}. If it's your first time, you may need to click through some initial setup prompts.
  2. Enable billing:
    Walk through the steps at console.cloud.google.com/billing{{ ext_link_icon() }} to enable billing. You'll need a credit card.
  3. Select a project:
    Either create a new project (click New Project in the top navigation dropdown), or, if this is your first time, just use the default ("My First Project").
  4. Enable the Cloud Vision API:
    In the left sidebar, go to APIs & Services → Library. Search for "Cloud Vision API" (it has a blue diamond logo). Click it, then click the blue Enable button.
  5. Create an API key:
    Go to APIs & Services → Credentials. Click + Create Credentials and select API key.
    A long string will appear. This is your API key. Copy it, store it securely, and treat it like a financial password. With this key, anyone can charge OCR processing (or other services, if you don't restrict the key) to your account.
  6. (Optional) Restrict the key:
    In the API key management screen, click the three dots next to your key, and choose Edit. Click Restrict key, and from the dropdown, select Cloud Vision API. Don't forget to click Save.
  7. Use the key:
    Return to the OCR page, select Google Cloud Vision, and paste your key into the Google Cloud API key field.

2. Sarvam Vision — written walkthrough
  1. Create an account:
    Go to sarvam.ai{{ ext_link_icon() }} and sign up.
  2. Add a payment method:
    Navigate to your account billing settings and add a credit card.
  3. Generate an API key:
    In your dashboard, find the API keys section and create a new key. Copy it and store it securely — treat it like a financial password.
  4. Use the key:
    Return to the OCR page, select Sarvam Vision, and paste your key into the Sarvam API key field.

How much will it cost?

Google Cloud Vision: The first 1,000 pages per month are free. After that, every additional 1,000 pages costs $1.50 ($0.0015/page). New accounts come with a $300 credit valid for 90 days. For light personal use you likely won't exceed the free tier. You can track your usage (e.g., see this Reddit post{{ ext_link_icon() }}).

Note: Google Cloud Vision uses standard Google Cloud billing (postpay — you're charged at the end of the month). This is separate from the Gemini API, which as of March 2026 requires prepay credits for lower-tier developer accounts. Don't confuse the two: Cloud Vision billing is set up through Google Cloud Console{{ ext_link_icon() }}, not AI Studio.

Sarvam Vision: ₹0.50 per page (~$0.006 as of May 2026). New accounts receive some free credits. In addition to the API (e.g. via Skrutable), you can also use the Sarvam Playground{{ ext_link_icon() }} directly — this does not require an API key but does limit each job to 5 pages.

Is this safe? Can I trust you/the provider?

Your API key is never stored by this application. The open-source code{{ ext_link_icon() }} reads the key from the HTML form, sends it with the OCR request to the provider, and discards it. Store your key with a password manager (e.g., 1Password) to keep it secure. If you have concerns about PDF contents, avoid uploading sensitive material — it is sent to a third-party cloud service.

How big can the input be?

Files up to ~{{ max_size }} MB are supported. Split larger files into parts. Google Cloud Vision handles up to 2,000 pages natively, so you'll almost always hit the file size cap before the page cap. Sarvam Vision's API accepts only 10 pages at a time, but this interface handles the splitting and reassembly automatically.

Will it work with complex page formats?

Maybe, maybe not. Cropping your PDF pages often improves both accuracy and performance. Sarvam Vision is designed with Indic script layouts in mind and appears to do better with multi-column Sanskrit pages.

Why are some characters wrong, words misplaced, etc.?

All OCR results contain some errors. Please review and clean up as needed, e.g., using regular expressions.

Why are lines returned as-is, e.g., with hyphenation?

These OCR options are literal. They returns what they see on the page, including line breaks and hyphens. This is another thing to review and clean up as needed.
