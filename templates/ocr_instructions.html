Why another OCR option?

Most Sanskrit students and scholars are not using truly high-quality OCR and don't know how to. This blog post explains the initial choice of Google Cloud Vision being promoted here. Meanwhile, other, larger AI models may now be taking the lead on performance, However, interfacing with them can be even more complex, so I'm still focusing on Cloud Vision for now. This interface takes (almost) all the difficulty out of interacting with Cloud Vision, including coding and cloud file management.

Why do I need an API key? How do I get it?

Because Google Cloud Vision isn't free, and because I don't yet have funding to foot the bill for everyone personally, I pass along the billing aspect to individual users. Specifically, an API key tied to a working billing account is required for this setup. An API key is basically just a sequence of characters that you keep secret, like a password. You can create and destroy as many as you like, at any time. Follow the video instructions below to set up your Google Cloud billing account, enable Cloud Vision for your account, and create an API key linked to the service.



For a verbal walkthrough, see below.

How much will it cost?

In most cases, virtually nothing. The first 1,000 pages per month are free. After that, every 1,000 pages is $1.50. You pay only for what you use. New accounts start off with a generous amount of credit ($300) that lasts 90 days. It's possible to track your usage (e.g. see see here on Reddit), but if you're doing personal projects, then chances are you won't exceed 1,000 pages a month.

Is this safe? Can I trust you/Google?

Yes, this is quite safe. You don't need to worry about Google doing things with the contents of your PDF. If you are concerned, then just don't upload items with sensitive information. As for me and my code, I promise that API keys are not being stored in any way. The code, which is entirely open source, simply reads the key from the HTML form (here), sends it along with the OCR request to Google (here), and that's it. I encourage you to use a password manager (e.g. 1Password) to store and fill the API key.

Detailed API Key Instructions

  1. Bring up Google Cloud Console:
    While logged into your Google account, go to the Google Cloud Console. If you've never used it before, you may need to click through some initial setup prompts.
  2. Enable billing:
    Walk through the steps at Billing to enable billing. You'll need a credit card, (for questions about cost, see above).
  3. Select project:
    Either create a new project (click New Project> in project dropdown in top nav...), or, if this your first time, leave it on the single default (“My First Project”).
  4. Enable Cloud Vision API:
    In the left sidebar, go to APIs & Services → Library. Search for “Cloud Vision API” (it has a blue diamond logo). Click it, then click the blue Enable button.
  5. Create an API key:
    Go to APIs & Services → Credentials. Click + Create Credentials and choose API key.
    A long string will appear — this is your API key. Copy it, store it, and treat it like you would any other financial password. With this key, anyone can charge OCR processing (or other services, if you don't restrict the key) to your account.
  6. (Optional) Restrict the key:
    On the API key management screen in the Google Cloud Console, click the three dots to the right of the relevant key, and choose Edit. Click Restrict key and select Cloud Vision API from the dropdown. Don't forget to Save.
  7. Use the key:
    You are now ready to return to the OCR page and paste your key into the input box labeled “Google Cloud API key.”
